🚀 GETTING STARTED
═══════════════════════════════════════════════════════════════════

1. SETUP ENVIRONMENT
   
   Create .env.local file with:
   CONTENTFUL_SPACE_ID=your_space_id
   CONTENTFUL_ACCESS_TOKEN=your_token
   
   (See CONTENTFUL_GUIDE.md for step-by-step)

2. INSTALL & RUN
   
   npm install
   npm run dev
   
   Access: http://localhost:3000

3. BUILD FOR PRODUCTION
   
   npm run build
   npm run start

═══════════════════════════════════════════════════════════════════

📝 CONTENT MANAGEMENT
═══════════════════════════════════════════════════════════════════

ADD ARTICLE (NO CODE NEEDED):
  1. Go to https://app.contentful.com
  2. Click "Create Entry" > "Article"
  3. Fill: Title, Slug, Date, Category, Body
  4. Click "Publish"
  5. ✅ Article appears on /stiri automatically

ADD PROJECT:
  1. Edit lib/projects.json
  2. Add new object to "projects" array
  3. Fill all fields
  4. Restart: npm run dev
  5. ✅ Project appears on /proiecte automatically

EDIT STATIC PAGES:
  1. Edit app/[page]/page.js
  2. Save file
  3. ✅ Changes appear automatically (hot reload)

💡 TIPS & TRICKS
═══════════════════════════════════════════════════════════════════

1. Hot Reload While Developing
   npm run dev - Changes save instantly

2. Static Generation for Speed
   Projects use SSG - builds once, serves fast

3. Rich Text in Articles
   Use HTML in Contentful body field:
   <h2>Title</h2>
   <p>Content with <strong>bold</strong> text</p>

4. URL-Friendly Slugs
   Use: "my-article-2024" NOT "My Article 2024"

5. Adding Related Projects
   In projects.json, use "relatedProjects": ["slug1", "slug2"]

🆘 SUPPORT & TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════

Articles not showing?
  → Check Contentful API token in .env.local
  → Verify token is Content Delivery API
  → Articles must be "Published" not "Draft"

Build fails?
  → npm install again
  → Delete node_modules and .next
  → Clear npm cache: npm cache clean --force

Images missing?
  → Check images are in public/images/
  → Use /images/filename.jpg NOT ../images/

Need help?
  → CONTENTFUL_GUIDE.md - Article questions
  → NEXTJS_MIGRATION.md - Technical questions
  → office@oncogen.ro - General inquiries
